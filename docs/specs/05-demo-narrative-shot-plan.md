# 05 Demo Narrative + Shot Plan

## Purpose
This document bridges the product and the later Remotion video. It defines the story the audience should experience, the order of reveals, and the visual beats the prototype must support.

## Core narrative
The demo should tell one clear story:

**Today’s H-1B to green card process is fragmented and opaque.  
A shared case system can reduce avoidable friction, catch issues earlier, and align applicant, employer, and attorney around one source of truth.  
It improves the process without pretending to solve structural scarcity.**

## Recommended demo length
Primary target:
- 75 to 120 seconds

Stretch version:
- up to 150 seconds

Avoid:
- over 2.5 minutes for the core walkthrough

## Story structure

### Beat 1: current-state tension
Goal: emotionally frame the problem.

Visual ideas:
- document stack
- fragmented portal labels
- unclear status snippets
- multiple handoffs
- “what happens next?” uncertainty

Possible voiceover:
“Right now, one case is spread across agencies, forms, emails, physical notices, and legal interpretation. No one sees the full picture.”

### Beat 2: introduce the future-state platform
Goal: show the portal as the technology centerpiece.

Visual ideas:
- clean reveal of homepage or overview
- strong headline around shared case record
- capability cards
- quick zoom into the flagship case

Possible voiceover:
“Our future state centers on one shared case record that connects applicant, employer, and attorney around the same information.”

### Beat 3: show the shared case workspace
Goal: embody the single source of truth.

Visual ideas:
- case header
- milestone timeline
- DOL / USCIS / DOS status view
- next-step card
- alerts panel

Possible voiceover:
“Instead of checking multiple systems, every stakeholder can understand the case from one unified timeline.”

### Beat 4: prove value with smart intake and validation
Goal: demonstrate causality and problem prevention.

Visual ideas:
- shared fields syncing to multiple forms
- readiness score
- one issue highlighted
- field update
- issue resolves
- score improves

Possible voiceover:
“The highest-leverage change is catching contradictions before filing. One update to the shared record can correct multiple downstream forms at once.”

### Beat 5: switch roles
Goal: show stakeholder alignment without duplication.

Visual ideas:
- segmented control switches from Applicant to Employer to Attorney
- same case, different priorities
- applicant sees plain-language next step
- HR sees deadline / cost / continuity risk
- attorney sees filing blockers and legal review

Possible voiceover:
“The same case record supports different decisions for different stakeholders, without creating different truths.”

### Beat 6: show deadline protection
Goal: prove operational value for employers and case managers.

Visual ideas:
- extension warning
- auto-trigger or reminder state
- task created
- risk lowered

Possible voiceover:
“Delay does not just create anxiety. It creates recurring cost and deadline risk. The system makes those risks visible earlier.”

### Beat 7: zoom back out to scope clarity
Goal: keep the argument honest and strong.

Visual ideas:
- concise comparison panel
- solves directly vs requires broader reform
- portal nested inside broader future-state ecosystem

Possible voiceover:
“This does not create new visa supply. But it does reduce rework, opacity, and coordination failure inside the process we have today.”

### Beat 8: close on impact
Goal: end with confidence and memorability.

Visual ideas:
- final metrics strip
- clean full-screen statement
- product logo / name
- one strong closing line

Possible voiceover:
“A shared case system cannot remove every structural constraint. It can remove avoidable friction, and that is where better process starts.”

## Recommended shot order
1. Current-state chaos framing
2. Overview / hero
3. Case workspace
4. Timeline focus
5. Intake sync
6. Validation issue appears
7. Field corrected
8. Readiness score improves
9. Role switcher: applicant
10. Role switcher: employer
11. Role switcher: attorney
12. Deadline alert / extension protection
13. Scope clarity / broader reform layer
14. Final impact close

## On-screen caption strategy
Use captions sparingly. Prefer short phrases:
- Fragmented today
- One shared record
- Catch issues earlier
- Same case, different roles
- Deadlines made visible
- Process help, not magic
- Better coordination, fewer surprises

## Visual pacing
- first 10 to 15 seconds: fast tension and framing
- middle: slower, more legible product proof
- end: crisp synthesis and close

## Key visual proof moments
These are the moments that must read instantly even without audio:
1. multiple forms syncing from one source
2. issue appears
3. issue resolves
4. readiness score improves
5. role changes alter the interface
6. one case is visible across multiple agencies
7. scope note prevents overclaiming

## Screen / state prerequisites for the app
The product must support:
- a clean overview state
- a clean case state
- a validation issue state
- a resolved issue state
- role-specific UI states
- an impact / scope state
- stable final frame

## Example voiceover script skeleton
Use this only as a directional draft.

“Today, the H-1B to green card journey is fragmented across agencies, forms, deadlines, and handoffs.  
No one sees the full picture, and small inconsistencies can create months of delay.

Our future state centers on one shared case record.  
Applicant, employer, and attorney are aligned around the same information, timeline, and next steps.

Here, shared intake data syncs across the filing package.  
Before submission, the system flags contradictions that would otherwise surface months later.  
Fix the issue once, and the entire case becomes more ready.

Because the record is shared, each stakeholder sees what matters most to them.  
Applicants get plain-language guidance.  
Employers get deadline and continuity visibility.  
Attorneys focus on legal judgment, not repetitive reconciliation.

This portal does not solve visa caps or retrogression.  
But it does reduce avoidable rework, opacity, and coordination failure.  
That is a meaningful future-state improvement.”

## Audio / soundtrack direction
If music is used:
- restrained
- modern
- subtle tension to clarity arc
- no trailer bombast

## Editing principles for the Remotion phase
- prefer clean hard cuts plus soft motion inside scenes
- let key interfaces breathe long enough to understand
- do not stack too many simultaneous callouts
- each shot should have one takeaway

## Ending frame options
Option A:
**One source of truth for a fragmented journey**

Option B:
**Reduce rework. Clarify status. Align stakeholders.**

Option C:
**A better process, even before broader reform.**

## Acceptance checklist
The demo narrative is successful if:
- a viewer understands the problem before the product appears
- the product proves one central idea rather than many unrelated ideas
- the role-based perspective change feels meaningful
- the scope note increases credibility rather than weakening the pitch
- the final impression is polished, realistic, and memorable
